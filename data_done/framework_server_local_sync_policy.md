# Framework Server / Local Sync Policy

## Source Of Truth

- Server is the source of truth for run artifacts: adapter directories, prediction JSONL, large generated data, and runtime summaries generated by GPU jobs.
- GitHub is the source of truth for code, configs, prompt templates, lightweight reports, and lightweight CSV/JSON summaries.

## What Goes To Git

Commit:

- Python scripts
- configs under `configs/framework/`
- prompt templates under `prompts/framework/`
- lightweight reports, CSV, and JSON summaries under `data_done/`

Do not commit:

- `data_done/*.jsonl`
- `data_done_lora/*.jsonl`
- prediction JSONL under `output-repaired/framework/`
- adapters/checkpoints under `artifacts/lora/`
- tar/zip bundles or raw logs

## Manifest Discipline

After every server run, generate:

```bash
python scripts/framework_artifact_manifest.py
```

This writes:

```text
data_done/framework_artifact_manifest_server.json
data_done/framework_artifact_manifest_server.md
```

The manifest records path, size, mtime, category, and SHA256 for files smaller than 50MB.

## Sync Discipline

When Codex local analysis needs a server artifact, sync only the required files. Do not sync all adapters or all prediction JSONL by default.

If local Codex cannot see a required server file, the correct status is `missing_server_artifact`. Do not infer results from old local files or copied console snippets.

## Preventing Stale Local Pollution

Framework stages should prefer explicit paths from the server manifest. Avoid broad auto-scans over old `output-repaired/` directories unless the stage explicitly asks for legacy observation results.

## Typical Flow

1. Run training/eval on server.
2. Generate manifest on server.
3. Commit lightweight summaries/reports if safe.
4. If Codex local audit needs predictions, tar only the specific prediction JSONL.
5. Never commit adapter weights or large JSONL.
