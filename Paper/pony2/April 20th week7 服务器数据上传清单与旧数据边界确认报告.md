服务器上的 `data` 目录目前不能继续作为 week7 主线实验输入使用。根据这次核对，服务器端现有内容更像早期误传到 GitHub 后同步过去的旧小数据，主要是少量根目录 pointwise test/noisy 文件和 Movielens 原始包，并不是当前 version pony 从 week5 到 week7 真正使用的 processed / multitask 数据。因此下一步不能在服务器当前 `data` 状态上继续跑 Qwen3 smoke、medium-scale 或 baseline matrix，也不能假设这些旧文件可以代表当前主线。正确路线是先从本地真实实验目录上传已经处理好的 `data/processed` 主线数据，不重跑 preprocess，不引入新变量，先把数据层补齐，再继续执行 Qwen3 parser/thinking 兼容验证、三任务 smoke config 和 batch 层运行。

本地真实可用的 pony 主线数据集中在 `D:/Research/Uncertainty-LLM4Rec/data/processed`。其中当前 week7 Qwen3 直接依赖的是 `data/processed/amazon_beauty`，它包含 pointwise、candidate ranking、pairwise 和 pairwise coverage 所需的完整输入：`train.jsonl`、`valid.jsonl`、`test.jsonl`、`ranking_valid.jsonl`、`ranking_test.jsonl`、`pairwise_valid.jsonl`、`pairwise_test.jsonl`、`pairwise_coverage_valid.jsonl`、`pairwise_coverage_test.jsonl`，以及 `items.csv`、`users.csv`、`interactions.csv`、`popularity_stats.csv`。这个目录本地约 68 MB，`test.jsonl` 有 5838 行，`ranking_test.jsonl` 有 973 行，`pairwise_coverage_test.jsonl` 有 500 行，`pairwise_test.jsonl` 有 4865 行。它才是当前 Beauty + Qwen3 本地后端 smoke、medium-scale、literature baseline 和后续 structured risk rerank 依赖链的最小可信数据源。

四域 compact pony 口径还需要另外三个 `_small` processed 目录：`data/processed/amazon_movies_small`、`data/processed/amazon_books_small`、`data/processed/amazon_electronics_small`。这三个目录不是旧 toy 文件，而是 week6 magic 7th 之后四域 compact replication 使用的处理后结果，分别包含 `train.jsonl`、`valid.jsonl`、`test.jsonl`、`ranking_valid.jsonl`、`ranking_test.jsonl`、`pairwise_valid.jsonl`、`pairwise_test.jsonl`、`items.csv`、`users.csv`、`interactions.csv`、`popularity_stats.csv`。Movies small 本地约 25.57 MB，Books small 本地约 45.27 MB，Electronics small 本地约 40.73 MB。把它们与 Beauty 一起上传后，服务器就不只是能跑当前 Beauty Qwen3 week7，还能自然承接后续 week8 的四域最小矩阵，不需要再因为缺少 processed/multitask 输入中断。

需要特别区分的是，`data/processed/test.jsonl`、`data/processed/test_noisy.jsonl` 和 `data/processed/test_noisy_metadata.json` 是根目录下 2KB 级旧小文件，不应被当成当前主线输入。`data/processed/movielens_1m` 虽然体量很大，但属于旧 Movielens 分支，不是当前 Amazon pony 主线。`data/processed/amazon_books`、`data/processed/amazon_electronics` 和 `data/processed/amazon_movies` 目前主要是大规模中间 CSV 或部分 jsonl，不具备当前 week5-week7 所需的完整 `ranking_*` 与 `pairwise_*` multitask jsonl 集合，因此也不应该被优先上传来跑当前服务器实验。noisy Beauty 系列目录可以后续 robustness 阶段再传，本轮 Qwen3 smoke 和 week7 medium-scale 不依赖它们。
