**week5-week6 Part5阶段完整收口总结**

到本轮最终升级完成之后，Part5 已经可以被准确表述为一个完整的 compact evidence version。这里的完整，不是说它已经替代后续 week7 之后的大规模验证，也不是说当前 100-sample real API 结果已经足以支撑最终顶会主实验结论，而是说 Part5 作为一个阶段，已经把研究对象、任务结构、方法主线、机制边界、同任务 baseline、论文级图表、四域紧凑复制和 paper-ready 主表骨架全部闭合。换句话说，week5 和 week6 不再是一组零散日报，也不再只是“多写了几个脚本”，而是已经形成一个可以直接支撑论文方法部分和初版实验叙事的阶段版本。

week5 完成的是结构底座。旧版项目的核心链条是 pointwise diagnosis、calibration 和 reranking，这条线仍然重要，但在 version pony 中已经不再被当成最终任务主体，而是被重新定位为 uncertainty diagnosis layer。week5 的真正贡献，是把 pointwise、pairwise preference 和 candidate ranking 三层任务放进同一套数据、配置、prompt/parser、推理、评价和 summary 结构中。pointwise 负责观察 confidence 是否可用、是否 miscalibrated、是否存在 popularity/exposure bias；pairwise preference 负责观察局部偏好机制和 uncertainty reliability；candidate ranking 负责承载真正接近推荐场景的列表级主决策。到 week5 结束时，这三层任务都已经拥有可生成的输入、可解析的输出、可计算的指标和可聚合的 summary，这就是后续 week6 方法迁移能够成立的前提。

week6 完成的是方法闭环。uncertainty 不再只停留在 pointwise calibration 后的分析变量，而是被推进到 candidate ranking 和 pairwise-to-rank 两条决策路径中。ranking decision family 从简单 linear penalty、coverage-aware、top-k gated，推进到 structured risk、local margin swap 和 fully fused 的统一 compare 空间；其中 structured risk 被收敛为当前默认主线，local swap 与 fully fused 被保留为 retained exploratory family，不删除、不抹掉，但也不再抢默认主实验资源。pairwise-to-rank 从局部成对准确率推进到能够反哺 ranking 的机制层路径，并通过 weighted aggregation 进入 ranking 指标空间。week6 还补齐了 estimator compare、same-task baseline、Part5 final results 和 final summary，使 Part5 不只是“方法可跑”，而是已经具备同任务对照和论文级收口。

magic 6 和 magic 7 补的是证据形态和跨域承接。magic 6 的重点不是新方法，而是 figure pack 与 coverage evidence：pointwise diagnosis 图说明为什么 raw confidence 不能直接当成概率；family compare 图说明 structured risk 为什么是当前默认主线，同时保留 local swap 和 fully fused 的探索身份；pairwise coverage 图说明 pairwise 有机制层正向信号，但低覆盖时不能被写成主决策替代。magic 7 则把 Beauty 单域闭环推进到 Movies、Beauty、Books、Electronics 四域 DeepSeek compact matrix，每个域保持 100-sample 左右的统一口径。这个矩阵的作用不是最终证明跨域稳定，而是回答一个更基础的问题：当前 Part5 结构是否能够离开 Beauty + Qwen 的单点设置，在四个正式数据域下维持统一执行和统一汇总。

本轮最终升级补的是四个最关键的阶段缺口。第一，四域 structured risk current best family 已经真正落地，不再只有 direct candidate ranking 的跨域复制；`outputs/summary/week6_final_4domain_structured_risk_compare.csv` 明确比较了 Movies、Beauty、Books、Electronics 上 direct ranking 与 structured risk 的 NDCG/MRR、改序幅度和 uncertainty coverage。结果显示 structured risk 在当前 compact setting 下整体是低扰动主线，Beauty 有小幅收益，其余域多数保持 direct ranking 不被破坏，这符合它作为稳健 current best family 的当前定位。第二，pairwise coverage 得到了实质修复：通过 coverage-balanced pair generation，Beauty compact setting 的 supported ranking events 提升到 100/100，但 pairwise_pair_coverage_rate 仍然显示每个 event 内部候选对覆盖有限，因此 pairwise 现在可以更扎实地写成机制层证据，仍不能过度写成主决策替代。第三，literature-aligned baseline 从入口存在推进到有初步防守力的 baseline 组，candidate-order、popularity-prior、longtail-prior、direct ranking 和 structured risk 被放进同一张 compare 表。第四，consolidated tables 已经把 single-domain Part5 主表、四域 DeepSeek 主表和 pairwise boundary 主表压成论文写作骨架，避免结果继续分散在十几张中间表里。

当前主线、保留线和 pairwise 边界必须写得很清楚。structured risk 是 current best ranking family，因为它在统一 compare 中表现为最适合继续承担默认主线的低扰动、coverage-aware、position-sensitive family；它不是因为某一次结果大幅刷分而成为主线，而是因为它比简单 penalty 更适合承担后续可复现验证。local swap 和 fully fused 不是失败后被删除的线，而是 retained exploratory family：它们继续留在代码、结果表、图和文档中，用于后续更大样本、更高 coverage、更复杂候选集或 robustness 场景下观察是否重新显出价值。pairwise 当前的边界也很明确：coverage 修复后，它已经比原先更有资格作为机制层证据进入论文叙事，但 plain 与 weighted 在本轮 compact setting 下 gap 为 0，说明这次最主要的收益来自事件支撑修复和局部偏好信号本身，而不是 uncertainty weight 已经稳定产生额外收益。因此，pairwise 仍然不能被升级为 candidate ranking 主任务替代线。

这一阶段没有把真实 API 样本从 100 扩到 1000，是一个有意的研究边界。当前任务不是用 API 成本堆出更大的表，而是在进入 week7 之前把 Part5 的 compact evidence 做完整、做透、做可追溯。如果现在直接扩到 1000，会把执行成本、API 速率和潜在不可复现性提前引入 Part5 收口阶段，反而掩盖当前真正需要解决的问题：任务是否统一、方法是否收敛、coverage 边界是否清楚、baseline 是否对齐、主表是否 paper-ready。更大规模、更强后端和更稳定批处理应该交给 week7 及之后的执行层升级，而不是在 Part5 最终收口时重新打开大规模实验发散。

当前 compact evidence version 已经足够支撑三类内容。第一，它足够支撑论文方法部分：三层任务定义、uncertainty 跨任务迁移、structured risk current best family、pairwise mechanism layer 和 baseline 体系都已经有对应实现。第二，它足够支撑论文实验初稿：有 Part5 final table、figure pack、四域 compact matrix、baseline compare 和 consolidated tables。第三，它足够支撑 reviewer-facing 的边界说明：pairwise coverage 之前的短板已经被正面处理，fully fused 没有被粗暴删除，100-sample 范围没有被伪装成最终大规模结论。它还不足以支撑的，是最终大规模稳定性、强 literature baseline 全量胜出、多模型多后端统一结论和 noisy robustness 的最终 claim，这些应该进入 week7 之后继续压实。

因此，做完这一轮之后，Part5 可以直接切 week7。切换的意思不是丢掉 Part5，而是停止在 Part5 内部继续发明 family 或补基础结构，把已经完成的 structured risk 主线、retained exploratory family、pairwise boundary、baseline compare 和 consolidated tables 带入更强执行层。week7 应该做的是执行环境、批量调度、强 baseline、可复现后端和更稳验证矩阵，而不是回头问 Part5 的任务结构和方法闭环是否成立。现在答案已经明确：在 100-sample real API compact scope 下，week5-week6 已经完成了一个完整、可解释、可防守、可衔接后续工作的 Part5 阶段版本。
