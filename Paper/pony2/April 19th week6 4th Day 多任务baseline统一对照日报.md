**April 19th week6 4th Day 多任务 baseline 统一对照日报**

今天这一天真正补上的，不是再增加一个零散脚本，而是把 `version pony` 在 Part5 阶段一直缺的一层实验结构正式补齐了。前面三天，我已经完成了 uncertainty 从 pointwise 诊断层向 `candidate_ranking` 与 `pairwise_to_rank` 的跨任务迁移，也已经建立了 ranking family compare、pairwise 机制线 compare 与 multitask uncertainty source compare。但如果停在这里，整套方法仍然容易被批评成“方法内部自己和自己比”，而不是在同一任务、同一输入、同一指标口径下和一个不使用 uncertainty 的直接对照对象做比较。今天 week6 第四天要解决的，就是这个问题。我不是去做 Part6 那种 literature-aligned 的强 baseline，也不是为了好看多拼几条线，而是先把 Part5 口径下必须有的 same-task baseline 层补齐，让多任务框架从“内部自洽的方法结构”推进成“具备直接对照能力的实验结构”。

工程上，今天新增了两个核心文件。第一个是 `src/methods/baseline_ranker_multitask.py`。这个文件承担的不是训练一个新模型，而是把 day4 需要的 baseline 逻辑从概念变成稳定、可复用的代码。它做了三件关键事情。第一，它把 pointwise 旧链里的 calibrated confidence baseline 和 uncertainty-aware rerank baseline 统一整理成 day4 compare 可直接接入的标准行，不再让 pointwise 层只以诊断表的形式存在。第二，它把 pairwise 这条机制线真正补出一个 plain aggregation baseline，而不是继续只拿 weighted pairwise-to-rank 和 direct ranking 去比较。我在这里没有偷懒用 direct ranking 代替 pairwise baseline，而是显式复用了现有 `uncertainty_pairwise_aggregator.py` 的聚合逻辑，把 `pair_reliability_weight` 强制改成 1.0，构造出真正不使用 uncertainty weight 的 `plain_win_count` 基线，再同时给出 overlap subset 和 expanded fallback 两种评测范围。第三，它把这些 baseline 行显式补上 `method_family`、`method_variant`、`evaluation_scope`、`is_same_task_baseline`、`is_current_best_family` 等字段，保证后面 week6 day5 聚合总表时，不会再出现“这条线到底是不是 baseline”“完全融合版是不是已经消失”这种叙事边界不清的问题。

第二个新增文件是 `main_baseline_multitask.py`。它是今天真正对外产生产物的主入口。这个脚本不是重新发明 compare 口径，而是明确建立在 day3 已经形成的 `outputs/summary/week6_day3_estimator_compare.csv` 之上，把已经存在的 current structured risk 主线、local swap、完全融合版、pairwise weighted aggregation、direct ranking reference 这些路径与今天新补出的 pointwise same-task baseline、pairwise plain aggregation baseline 汇成一张真正面向 Part5 的 baseline compare 总表。也就是说，今天这条主入口并不是“把所有结果简单拼接”，而是做了一个有意识的实验结构收紧：候选 ranking 主任务这边，我保留 direct ranking baseline、structured risk current best family、local swap retained family、完全融合 retained family；pairwise 机制线这边，我保留 direct overlap reference、plain win-count overlap baseline、weighted overlap method、direct expanded reference、plain expanded baseline、weighted expanded method；pointwise 层则补进 calibrated confidence baseline 与 uncertainty-aware rerank 同任务对照。这样一来，day4 得到的就不是一张散乱结果表，而是一张明确体现“哪一条是 same-task baseline，哪一条是 uncertainty-aware method，哪一条是 current best family，哪一条是 retained complex family”的统一矩阵。

今天最重要的产物是 `outputs/summary/week6_day4_decision_baseline_compare.csv`。这张表一共 12 行，但每一行都有非常明确的身份。pointwise 层里，我保留了 `pointwise_calibrated_confidence_baseline` 与 `pointwise_uncertainty_aware_rerank` 两条 same-task compare 路径，而且它们都统一挂在 calibrated 口径下，因此对应的 `ECE = 0.0808`、`Brier = 0.1589`、`AUROC = 0.8324` 不再误用 raw verbalized confidence 的诊断值。candidate ranking 层里，`direct_candidate_ranking` 作为明确的 same-task baseline 被保留下来，而 `structured_risk_family` 继续被标成 `is_current_best_family = True`，也就是当前默认主线。与此同时，`local_margin_swap_family` 与 `structured_risk_plus_local_swap_family` 也继续留在表里，明确标注为 retained family，而不是因为还没成为当前主线就被从 baseline compare 中抹掉。pairwise 层里，我今天真正补上的关键对照，是 plain aggregation baseline。现在 overlap subset 上的 `pairwise_plain_win_count` 达到 `NDCG@10 = 0.6905`、`MRR = 0.5972`，expanded fallback 后是 `0.6621 / 0.5550`。这两行非常重要，因为它们第一次让 pairwise 机制线内部有了真正的 same-task baseline：weighted pairwise-to-rank 现在不是只和 direct ranking 比，而是也要和“不使用 uncertainty weight 的同一聚合逻辑”去比较。

如果只看今天这张总表，本轮结果已经能给出几条相当清楚的判断。第一，pointwise 这一层的 same-task baseline compare 说明，旧版管线并没有因为现在转向多任务就失去价值。它依旧是 version pony 的诊断与 calibrated decision 起点，而且现在已经能被 clean 地放进多任务 compare 结构里，而不是游离在外。第二，candidate ranking 的主线收敛在 baseline 层被进一步写实了。`structured_risk_family` 虽然没有在 full-set 指标上大幅超过 direct ranking，但它是当前唯一既保持主线稳定性、又不在结构上恶化列表的 family，因此它继续作为 `current best family` 是合理的。第三，local swap 和完全融合版并没有在 day4 被挤出表外，相反它们被更清楚地定位了。当前最值得注意的是，完全融合版在 day4 的 baseline compare 中继续保留了与 local swap 相同的最好指标，也就是 `NDCG@10 = 0.6623`、`MRR = 0.5533`，而且这组数值来自 `consistency` source。这再次说明，完全融合版现在不应当成为默认主线，但也绝对不应该被删，因为它的“需要特定 uncertainty source 才显出局部优势”的身份已经越来越清楚了。第四，pairwise 机制线今天终于有了真正的 same-task baseline 结构，而且这个结构给出的结论是稳定的：weighted pairwise-to-rank 在 overlap subset 上仍然优于 plain win-count baseline，也优于 direct ranking reference；expanded fallback 之后，它也仍然轻微优于 plain baseline 和 direct reference。这使得 pairwise 机制线今天第一次不只是“比 direct ranking 好一点”，而是能够更有说服力地被解释成“uncertainty weight 在局部偏好聚合中确实带来了附加价值”。

如果把今天最重要的数字放在一起看，结论会更直观。candidate ranking 这边，direct baseline 是 `NDCG@10 = 0.6613`、`MRR = 0.5522`；structured risk 主线与它持平，但继续保持零改序；local swap 与完全融合版在最优 source 下都轻微上升到 `0.6623 / 0.5533`，但改序比例也随之上升，所以它们现在更像 retained family 而不是默认主线。pairwise overlap subset 这边，direct reference 是 `0.6863 / 0.5833`，plain win-count baseline 是 `0.6905 / 0.5972`，uncertainty-aware weighted pairwise-to-rank 是 `0.7029 / 0.6111`。这说明 pairwise 内部的 plain baseline 已经不是空对照，而是一条确实有一定排序能力的路径；但 uncertainty-aware weight 仍然能在它之上再把 NDCG 和 MRR 往上推一截。expanded full-set 这边，direct reference 是 `0.6613 / 0.5522`，plain baseline 是 `0.6621 / 0.5550`，weighted pairwise-to-rank 是 `0.6646 / 0.5578`。也就是说，coverage 限制依旧没有消失，但 uncertainty-aware 的 pairwise 机制收益在 fallback 扩展之后也没有消失。这比 day2 和 day3 的叙事更扎实，因为今天它不再只是“weighted vs direct”的比较，而是“weighted vs plain vs direct”三方同场比较。

今天这轮工作的研究贡献，我会写成四点。第一，我正式把 Part5 需要的 same-task baseline 层补齐了，让 pointwise、candidate ranking、pairwise 三层任务都拥有了清楚的“同任务、同输入、同指标，但不使用 uncertainty”的直接对照对象。第二，我让 pairwise 机制线第一次拥有了真正的方法内 baseline，而不是继续只拿 direct ranking 去做跨路径参照。第三，我把 current best family 与 retained family 的身份边界在 baseline 层写实了。现在完全融合版不会因为还没成为默认主线就从总表里消失，它被保留在统一 compare 里，同时通过 `is_current_best_family` 这种显式字段，避免和当前主线混淆。第四，我把 day4 的产物直接做成了后续 day5 总收口能无缝接住的格式。`week6_day4_decision_baseline_compare.csv` 的表头已经和前几天的 compare 表统一，因此 week6 最后一天再做 `part5_multitask_final_results.csv` 时，不需要重新回头修 schema。

今天结束时，我对 week6 第四天的结论会写得比较明确。Part5 的多任务框架现在已经不再只是一个方法草图，而是形成了 uncertainty-aware 与 non-uncertainty 的 same-task 直接对照实验体系。candidate ranking 这边，structured risk family 继续稳住当前默认主线；local swap 和完全融合版继续作为 retained family 留在统一 baseline compare 中，而不是被提前取消。pairwise 这边，plain aggregation baseline 的补齐使 weighted pairwise-to-rank 的优势第一次获得了真正的方法内解释空间，也就是我现在可以更有底气地说，pairwise 机制线的收益不只是来自“走了一条不同路径”，而是来自 uncertainty 确实在局部偏好聚合阶段提供了额外信息量。pointwise 这边，旧版 calibrated rerank 管线也已经被自然纳入多任务结构，而不是成为一个必须在论文里额外解释的历史残留。换句话说，今天 day4 做完之后，我手里的 Part5 已经从“多任务方法空间”进一步推进成了“多任务 + same-task baseline + current best family / retained family 身份清晰”的实验体系。这正是 week6 第五天能够真正收口成论文级总结之前，必须完成的一步。

如果说今天还有一个很值得后面继续盯住的建议，那就是 pairwise plain baseline 的存在其实让 week7-week8 的 reviewer 防御口径更强了。过去如果 reviewer 问“你看到的 pairwise 提升，会不会只是 aggregation 本身带来的，不是 uncertainty 带来的”，我只能用 direct ranking 参照来回答，这种回答仍然容易被追问。现在我已经有了 plain win-count baseline，这个问题后面就能被更扎实地回答。也就是说，今天补上的并不只是一个 baseline 文件，而是一个对后续审稿叙事很关键的防守层。
