**April 18th week6 3rd Day 多任务 uncertainty source compare 与 coverage 扩展日报**

今天真正完成的，不是再多跑几张表，也不是把旧版 `main_uncertainty_compare.py` 机械搬到多任务分支上，而是把 `version pony` 在 week6 第三天需要回答的核心问题第一次真正落成了统一证据层。前两天解决的是 uncertainty 能否进入 `candidate_ranking` 和 `pairwise_preference`，今天解决的则是一个更接近论文主张的问题：当 uncertainty 已经跨任务迁移之后，不同 uncertainty source 在不同决策粒度下到底表现如何，它们是否仍然保持同样的排序价值，以及 pairwise 机制线当前看到的局部优势究竟是可靠信号，还是只是在极小支持子集上偶然出现的现象。也正因为这个问题比前两天更像“系统比较”而不是“功能落地”，所以今天的工作重点不再是单独发明新公式，而是把已有 estimator、已有 ranking family、已有 pairwise-to-rank 路径全都压进统一 compare 口径，并把 coverage 问题直接纳入同一天的结果结构里，而不是留到后面当补丁处理。

工程上，今天新增的核心文件是 `main_uncertainty_compare_multitask.py`。它不是一个只服务单任务诊断的小脚本，而是一个显式面向 `pointwise_yesno`、`candidate_ranking`、`pairwise_to_rank` 三层任务的统一比较入口。这个脚本首先读取 `outputs/beauty_qwen/calibrated/test_calibrated.jsonl` 和 `outputs/beauty_qwen/self_consistency/test_self_consistency.jsonl`，通过 `src/uncertainty/estimators.py` 里已经存在的 estimator 逻辑，把 `verbalized_raw`、`verbalized_calibrated`、`consistency`、`fused` 这四类 uncertainty source 统一成可直接进入 compare 的标准列；随后它把这些 uncertainty source 分别送进 pointwise calibration view、ranking family compare 和 pairwise-to-rank compare。对 ranking 而言，我没有重新做一轮大范围参数搜索，而是直接调用 `outputs/summary/week6_rank_rerank_search_meta.json` 中 week6 day1 已经收敛出的当前参数，分别评估 `nonlinear_structured_risk_rerank`、`local_margin_swap_rerank` 和 `structured_risk_plus_local_margin_swap_rerank` 在不同 uncertainty source 下的表现。对 pairwise 而言，我继续沿用 day2 已经落地的 `uncertainty_pairwise_aggregator.py`，但这次不再满足于 overlap subset 的一张 compare 表，而是进一步加入 coverage 诊断和 coverage 扩展逻辑，把 pairwise 目前支持不到的 ranking event 显式用 direct ranking fallback 补齐，并把 overlap 结果与 expanded 结果同时记入统一 summary。换句话说，今天这套实现的目的不是让 compare 看上去更丰富，而是让不同 uncertainty source 真正进入“同一任务、同一 family、同一 coverage 解释框架”中被比较。

具体产物上，今天至少形成了两张关键表。第一张是 `outputs/summary/week6_day3_estimator_compare.csv`，它把 pointwise diagnosis、candidate ranking rerank family、pairwise-to-rank overlap subset、pairwise-to-rank expanded fallback 全都放进了同一 schema 里，列头中显式保留了 `domain`、`model`、`task`、`estimator`、`method_family`、`method_variant`、`evaluation_scope`、`HR@10`、`NDCG@10`、`MRR`、`ECE`、`Brier`、`AUROC`、`coverage`、`head_exposure`、`changed_ranking_fraction`、`avg_position_shift` 等字段。第二张是 `outputs/summary/week6_day3_pairwise_coverage_compare.csv`，它专门记录 pairwise 机制线当前的事件支持率、pair 覆盖率、平均可靠性权重，以及 overlap 与 expanded 两种评测范围下的 NDCG 和 MRR。今天我认为这两张表比“再额外跑一个方法”更重要，因为它们第一次让 `version pony` 真正具备了一个顶会风格的回答方式：我不仅能说某条线有效，还能说它有效在什么范围、依赖什么 uncertainty source、在 coverage 不足时应当如何解释，而不是把所有结果直接堆成一张没有边界感的 headline table。

今天最值得认真写进记录的，是数据本身已经开始给出很清楚的分层结论。先看 pointwise diagnosis，`verbalized_calibrated` 依然是当前最稳的 uncertainty source，它在 Beauty + Qwen 上给出的 `ECE = 0.0808`、`Brier = 0.1589`，明显优于 `verbalized_raw` 的 `ECE = 0.3706`、`Brier = 0.3616`，也显著优于当前这版 `consistency`。这说明旧版项目中“先把 verbalized confidence 校准，再把 uncertainty 用于后续决策”的主逻辑在多任务 setting 下没有被推翻，相反仍然构成最可信的诊断起点。再看 candidate ranking 这一层，今天最有信息量的结果不是 structured risk 有没有大幅提升，而是不同 uncertainty source 对不同 decision family 的作用方式开始出现分化。`nonlinear_structured_risk_rerank` 在四种 uncertainty source 下都与 direct ranking baseline 基本持平，仍然保持了它现在最重要的主线身份：稳定、不乱改序、至少不破坏原始排序质量。`local_margin_swap_rerank` 和完全融合版在 `verbalized_calibrated` 下表现更差，`NDCG@10` 和 `MRR` 都掉到 `0.6479 / 0.5322`，但在 `consistency` 下却轻微回升到 `0.6623 / 0.5533`，甚至略高于 direct ranking 的 `0.6613 / 0.5522`。这个结果非常关键，因为它第一次把“完全融合版为什么要保留并行观察”从主观直觉变成了数据事实。它现在仍然不应该抢当前主线，因为整体稳定性不够，而且在更可靠的 `verbalized_calibrated` source 下并没有赢；但它也确实不该被提早删除，因为一旦 uncertainty source 发生变化，它就已经开始出现局部反超信号。

比 ranking family 更醒目的，是 pairwise-to-rank 这一层在不同 uncertainty source 下的分化更明显了。在 overlap subset，也就是当前真实有 pairwise 支持的 `6/30` 个 ranking event 上，`verbalized_calibrated` 给出的 `pairwise_to_rank_weighted_win_count` 达到了 `NDCG@10 = 0.7029`、`MRR = 0.6111`，明显高于 direct ranking 在同一交集子集上的 `0.6863 / 0.5833`，也高于其他 uncertainty source。`fused` 次之，`verbalized_raw` 和 `consistency` 更弱。这一点说明 pairwise 机制线当前的优势并不是“只要聚合 pairwise 就一定更强”，而更像是“当局部偏好边的可靠性权重由更稳定的 calibrated uncertainty 提供时，pairwise 才真正开始释放细粒度偏好价值”。这其实非常符合项目总览里对三层任务结构的定义。pointwise 并不是要自己取代主任务，而是负责产出一个更可信的 uncertainty source；当这个 source 进入 pairwise 这种更细粒度的局部偏好机制层时，它比直接进入 current ranking rerank 可能更容易显出额外信息量。这是今天非常值得后面继续深挖的一条研究信号。

但今天同样重要的一点，是我没有因为这组数值好看就回避 coverage 现实。`week6_day3_pairwise_coverage_compare.csv` 已经把当前 pairwise 机制线的证据边界写得非常明确：不管使用哪一种 estimator，目前 `supported_event_count` 都只有 `6`，`total_ranking_event_count` 是 `30`，也就是 `pairwise_supported_event_fraction = 0.2`；平均每个被支持事件只有 `5` 个 pair，整体 `pairwise_pair_coverage_rate = 0.3333`。这意味着如果我只拿 overlap subset 的最好结果去说 story，就会高估 pairwise 线的成熟度。所以今天我没有暂停 day3，也没有把 coverage 问题推给 day4，而是直接把 coverage 扩展并进 compare：对没有 pairwise 支持的 ranking event，我用 direct ranking 结果做显式 fallback，得到 expanded 版本的整套 ranking 输出。这样做以后，`verbalized_calibrated` 的 pairwise-to-rank 仍然略高于 full-set direct ranking，expanded 后达到 `NDCG@10 = 0.6646`、`MRR = 0.5578`，而 direct ranking full set 是 `0.6613 / 0.5522`。也就是说，pairwise 线在 coverage 扩展之后优势明显缩小了，但没有完全消失。这比 overlap subset 的 headline 更可信，也更适合后续论文叙事，因为它把“机制线局部优势存在”与“当前覆盖仍然不足”这两个事实同时保留下来了。

如果把今天的研究贡献压缩成几条清楚的话，我会这样写。第一，我把旧版单任务 uncertainty source compare 正式升级成了多任务 compare，`version pony` 第一次可以在同一张表里同时看 pointwise calibration 质量、ranking rerank family 效果和 pairwise-to-rank 机制层效果。第二，我把 day2 暴露出来的 pairwise coverage 问题直接纳入了 day3 的方法与结果结构里，而不是把它留成一个后续需要口头解释的漏洞；这一步让 pairwise 证据第一次具备了“局部优势”和“全局支持范围”同时可见的研究诚实性。第三，我让“完全融合版是否值得继续保留并行观察”这个问题第一次拥有了明确的数据触发依据。现在的数据并不支持它升级为当前默认主线，但确实支持它继续保留，因为它在 `consistency` source 下已经出现轻微反超，而这说明它更像一个依赖 source 条件的高复杂度候选 family，而不是应当立即删除的失败版本。第四，我把 pointwise、ranking、pairwise 三层任务真正连成了方法闭环。今天的结果非常直接地说明，不同 uncertainty source 在不同决策粒度下并不等价，`verbalized_calibrated` 更像当前最好的通用诊断 source，而 pairwise 机制层对 calibrated uncertainty 的收益放大更加明显。

今天结束时，我对 week6 第三天的结论会写得比较克制但也比较清楚。第一，`version pony` 现在已经不再只是“多了几个任务”，而是开始具备系统研究 uncertainty 如何定义、如何迁移、如何利用的真正比较能力。第二，当前 candidate ranking 主决策层的默认主线仍应保持为 structured risk family，因为它在不同 uncertainty source 下都最稳，至少不破坏 direct ranking 的原有质量；local swap 和完全融合版继续保留，但目前只能作为待观察 family，而不是 current best family。第三，pairwise-to-rank 已经在 overlap subset 上形成了可信的积极信号，尤其在 `verbalized_calibrated` source 下，它表现出不同于 direct ranking 的细粒度偏好价值；但这条线当前还只能被定位为 mechanism line 和并行验证线，而不能越级改写主线，因为 `6/30` 的事件支持率还远远不够。第四，从下一步推进价值上看，最值得投入的不是继续增加更多 pairwise 聚合公式，而是尽快提高 pairwise 与 ranking 的对齐覆盖，并继续观察 `verbalized_calibrated` 与 `consistency` 这两类 uncertainty source 在复杂 family 上是否会进一步拉开分工。只有当 coverage 扩大以后，pairwise 机制线今天呈现出的优势，才真正有资格进入 week7 之后更强级别的主线竞争讨论。
