# From Teacher Line to SRPD Bridge

## Why this bridge exists

The original teacher-requested line asked for a server-side local 8B plus LoRA small-model execution path that can replace the practical execution role previously carried by the official API line.

The project later expanded into:

- official API observation and teacher evidence
- structured-risk hand-crafted ranking and rerank
- SRPD trainable uncertainty-aware enhancement
- Week8 outer compare
- Week9 system integration

Without an explicit bridge, these lines can look disconnected.

## What Week7.8 repairs

Week7.8 does not replace the later research lines. It fills a missing execution layer:

- official API remains the observation and teacher-evidence route
- local-v2 becomes the replay execution route
- structured risk remains the strongest hand-crafted baseline
- SRPD remains the higher-layer trainable enhancement family

## Three-layer interpretation

### Layer 1: Official API observation

This line is responsible for:

- historical observation
- early uncertainty signals
- teacher reliability evidence
- case studies
- later outer-compare reference

### Layer 2: Local-v2 replay mainline

This is the Week7.8 line. It is responsible for:

- replaying Week1-Week4 concerns
- using the current task structure rather than legacy yes/no framing
- executing on server-side local 8B plus LoRA
- becoming the teacher-requested main local execution line

### Layer 3: SRPD and system enhancement

This line is responsible for:

- uncertainty-weighted student learning
- gap-gated correction control
- direct-anchor fallback
- preference repair
- later system integration

## Why Week7.8 does not conflict with Week8 or Week9

Week7.8 is not a replacement for:

- Week8 fairness-aware outer compare
- Week9 uncertainty-aware SRPD system integration

Instead, it stabilizes the foundation under them. Once the teacher-requested local replay mainline is formalized, Week8 and Week9 can be written on top of a cleaner execution story.

## Current bridge conclusion

The intended project logic is:

1. preserve official API evidence
2. formalize local-v2 as the replay main execution route
3. keep structured risk as the strongest hand-crafted baseline
4. keep SRPD as the trainable enhancement family
5. move to Week8 compare and Week9 system on top of this clarified stack
